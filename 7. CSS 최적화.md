## CSS 최적화 :불필요한 CSS제거

### 탐지

- 개발자모드 > Lighthouse > Opportunities > Reduce unused CSS : 사용하지 않는 CSS를 제거하면 성능에 영향을 줄 지 파악해줌
- 개발자모드 > Coverage : 페이지에서 사용하는 js, css리소스에서 실제 실행하는 코드가 얼마나 되는지, 비율 어느정도인지 파악. Usage Visual..어쩌구에서 빨간색 비율이 많으면 미사용하는 부분이 많다는 뜻
- 주의
    - js는 if문 등의 분기 처리가 있어 해당 분기 안으로 ‘아직’ 접근한 적 없는 경우 미사용으로 체크될 수 있음 → js최적화는 비추
    - css 파일인데 빨간 막대가 많다면 최적화 고려 가능
- PurgeCSS 프로그램 사용 : 파일의 모든 키워드를 추출해 해당 키워드를 이름으로 갖는 CSS 클래스만 보존하고 나머지는 삭제
    - 최적화 후 기존 css 파일을 덮어쓰는 식으로 대체 가능
    - :(콜론) 등의 특수문자가 포함된 css를 하나로 인식 못하고 문자열 잘라버려서 삭제해버리는 수가 있음
        - `defaultExtractor` 옵션에서 키워드 추출 기준을 정규식으로 작성해서 좁힐 수 있음
