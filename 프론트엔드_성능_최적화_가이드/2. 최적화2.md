## 컴포넌트 지연로딩(lazy-loading)

컴포넌트를 분할 (코드 분할처럼)해서 컴포넌트가 쓰이는 순간에 로드. 이렇게 해서 이 컴포넌트에서만 쓰이는 라이브러리도 분리 가능

```jsx
import React, {useState, Suspense, lazy} from 'react'
// import ImageModal from './components/ImageModal' -> 일반로딩

const LazyImageModal = lazy(() => import('/components/ImageModal'))
```

- 정적으로 import되어 청크파일에서 분리됨
- Suspense : 분리한 ImageModal이 로드되기 전 발생하는 에러 방지
- 사용

```jsx
function App(){
	const [showModal, setShowModal] = useState(false);
	
	return(
		<div> ...
			<Suspense fallback={null}> <!-- ImageModal컴포넌트 로드 전에는 null처리 --> 
				{showModal ? (<LazyImageModal closeModal={()=>...}/> ) : null }
			</Suspense>
		</div>
	);
}
```

## 이미지 지연 로딩
바로 필요한 이미지가 아니라면 로딩을 뒤로 미루는 것
- 이미지 로딩 시점 결정 중요
- (주의) 스크롤 감지 기능 추가 시 로직 실행 매우 잦아짐  
  - Intersection Observer 사용으로 해결 가능
### InterSection Observer
- 브라우저 제공 API
- 웹 페이지의 특정 부분을 관찰 상태로 만들고, 해당 요소가 화면에 들어왔을 때만 함수 호출
- 옵션값
	1. root : (기본 null) 대상 객체의 가시성 확인. 기본값이면 브라우저 뷰포트로 설정
	2. rootMargin : (기본 '0px') root요소의 여백. root의 가시범위를 가상으로 확장 또는 축소가능
	3. threshold : (기본 1.0) 
- 리액트에서 useEffect에서 쓰는 이유 : 안하면 렌더링때마다 인스턴스가 생성. 수많은 observer가 관찰하게 됨
- 이미지 로딩은 img태그에 src가 할당되는 순간 일어나므로 콜백 실행 시 url할당하는 식으로 지연 로딩 처리

```javascript
function ImgCard(props) {
	const imgRef = useRef(null)
		
	useEffect(()=>{
		const options = {
			root : null, // 옵션1
			rootMargin : '0px', // 옵션2
			threshold : 1.0, // 옵션3	
		}
		
		//
		const callback = (entries, observer)=> {
			if(entry.isIntersecting) { // 화면 안으로 들어오는지 나가는지(false)여부
				entry.target.src = entry.target.dataset.src // src 주소 할당
				observer.unobserve(entry.target)
			}
		} 
		
		//원하는 요소 관찰할 옵저버 
		const observer = new IntersectionObserver(callback, options)
		
		//observer.observe(document.querySelector('#someId'))** // html에서 쓸때
		observer.observe(imgRef.current)
		
		return () => observer.disconnect()
	}, [])
	
	//---------
	return (
		<div>
			<img src={props.image} ref={imgRef}/>
			<div calssName='생략'>{props.childred}</div>
		</div>
	)
**}**
```

## 컴포넌트 사전로딩

컴포넌트 지연로딩 시 컴포넌트 호출이 느려지는 느낌을 받기 때문에 최초 접속시는 아니지만, 컴포넌트 호출 전까지는 로딩을 완료해두는 것

### 로딩시점1. 버튼 hover

300~600밀리초 정도로 로딩이 충분한 경우

```jsx
const handleMouseEnter =()=>{
	const component = import('./components/ImageModal'); // 여기서 미리 로딩
}

return (
	<ButtonModal
		onClick={()=>{setShowModal(true}}
		onMouseEnter={handleMouseEnter}
	> 버튼에 들어가는 내용쓰
	</ButtonModal>
)
```

### 로딩시점2. 모든 페이지 로딩 완료 & 컴포넌트 마운트 완료 시

로딩에 시간 소요가 좀 있는 경우

- 클래스형 컴포넌트 : `componentDidMount`
- 함수형 컴포넌트 : `useEffect`
    
    ```jsx
    function App(){
    	...
    	useEffect(() =>{
    		const component = import('./components/ImageModal')
    	},[])
    	...
    }
    ```
    

## 이미지 사전 로딩

필요한 시점보다 먼저 다운로드해 두고 필요할 때 바로 보여줄 수 있도록 하는 이미지 로딩 기법

이미지를 사용하는 시점에 로드하는 기존의 HTML, CSS방식 말고 자바스크립트에서 Image객체를 사용해서 로드 

```jsx
const img = new Image();
img.src = '{이미지 주소}' ; // 해당 코드가 실행되는 시점에 이미지 다운로드
```

- 응용 : 시점을 잘 파악해야됨. 적절한 시점에 코드를 심어줄 것. 예시는 모달 오픈이라 useEffect에 했음

```jsx
useEffect(() => {
	const component = import('./components/ImageModal')
	
	const img = new Image();
	img.src = '이미지 경로..'
} ,[])
```

## 브라우저 렌더링 과정

= 픽셀 파이프라인

1. 리소스 다운로드 (HTML파일, CSS파일 등) & DOM 생성
2. 렌더 트리(render tree) 생성 :  DOM + CSDOM
    - 화면에 표시되는 각 요소의 레이아웃 계산에 사용
    - `display: none` 은 표시되지 않음 (`opacity : 0` 은 표시됨)
3. 레이아웃 : 화면 구성 요소의 위치나 크기 계산. 해당 위치에 요소 배치
4. 페인트: 화면 배치 요소에 색 채우기
5. 컴포지트(Composite) : 각 레이어 합성. 화면을 그리기 위해 쪼갠 레이어 합성 단계

## 이미지 사이즈 최적화
- 체크 툴 : Light house - Oppertunities - properly size images
- 이미지 사이즈가 적절하게 호출되고 있는지 확인하는 법
    -  개발자도구- Elements- Rendered size가 Intrinsic size보다 많이 작으면 필요 이상으로 큰 이미지 사이즈로 호출하고 있는 것. 이미지 CDN을 활용해 줄여서 불러오거나, 로컬 이미지라면 사이즈를 줄여놓는 것이 좋음
- [Squoosh](../0-2.%20기타%20사용%20가능한%20툴들.md#Squoosh)를 사용해 이미지 변환

### 이미지 CDN
CDN(물리적 거리의 한계를 줄여놓는 기술, Content Delevery Network)에 이미지 특화 기능이 추가된 것. 이미지 크기 등 가공 기능이 추가됨. 
- 이미지 호출 URL에 ?width=사이즈&height=사이즈 등을 추가해서 호출 시점부터 크기를 줄여서 요청할 수 있음
